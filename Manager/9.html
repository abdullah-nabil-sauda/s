3 أنظمة فرعية في مشروع واحد

كل حدث في Trello يُعالج بذكاء

النظام يتكون من 3 مسارات عمل تعمل معاً: إنشاء البطاقات، إشعارات الأعضاء، وتنبيهات المواعيد.

نظام #1

إنشاء البطاقات والمزامنة

عند إنشاء بطاقة جديدة في Trello، يتم تسجيلها تلقائياً في Supabase وإنشاء Webhook مخصص لمراقبة تحديثاتها المستقبلية.

Trello Trigger → إنشاء بطاقة
حفظ في Supabase (cards)
إنشاء Webhook للبطاقة
مزامنة المواعيد والبيانات
نظام #2

إشعارات الأعضاء

عند إضافة أو إزالة عضو من بطاقة، يُرسل رسالة Telegram مخصصة ومرحة مع تفاصيل المهمة — ويُحدّث قاعدة البيانات تلقائياً.

Webhook → كشف نوع الحدث
فحص/إنشاء المستخدم
ربط المستخدم بالبطاقة
رسالة Telegram مرحة 🎉
نظام #3

تنبيهات المواعيد النهائية

كل 12 ساعة، يفحص جميع البطاقات المستحقة اليوم ويرسل تنبيه عاجل لكل عضو مُعيّن عليها عبر Telegram.

Schedule Trigger (كل 12 ساعة)
بطاقات مستحقة اليوم
جلب أعضاء كل بطاقة
تنبيه عاجل ⚠️ عبر Telegram

هل فريقك ينسى المواعيد النهائية؟

44% من الفرق تفوّت مواعيد التسليم لأن لا أحد يتابع البطاقات في Trello.

قبل الأتمتة

  • لا أحد يعرف أنه مُعيّنTrello لا يُشعر الأعضاء بشكل فعّال
  • مواعيد نهائية تمر بصمتلا تنبيهات حقيقية قبل أو عند الموعد
  • بيانات غير موثقةلا سجل مركزي للمهام والأعضاء

بعد الأتمتة

  • إشعار فوري على Telegramرسالة مرحة ومخصصة لكل عضو
  • تنبيه عاجل للمواعيدكل 12 ساعة — لا مهمة تُنسى
  • توثيق في Supabaseكل بطاقة، عضو، وربط — موثق تلقائياً

دليل الإعداد

1

إعداد Trello API

احصل على API Key و Token من Atlassian Developer ↗

ضع Board ID في Trello Trigger node (افتح بطاقة، أضف .json للرابط، ابحث عن "idList")
2

إعداد Supabase

أنشئ 3 جداول في Supabase ↗:

cards
id, card_name, board_name, due_date, url, completed
users
id, name
card_user
cardId, userId
3

إعداد Telegram Bot

أنشئ بوت عبر @BotFather في Telegram، انسخ Token، واضبط Chat ID في nodes الإرسال.

4

خصّص أسماء المستخدمين

في nodes "username" و "usernames 1"، اربط أسماء Trello بمعرّفات Telegram:

Username Mapping
"أحمد": "@ahmed_tg",
"سارة": "@sara_tg",
"خالد": "@khaled_tg"
